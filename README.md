barabaka
=====

Реализация тестового задания.
Коротко

- ходим за данными в эндпойнт
- реазизованы эндпойнты на самом сервисе для управления таймером и отдачей данных по запросу


Версия эрланг 28.

Запуск

если для доступа до тествого эндойнта нужен сокс5 прокси, то запускаем так:

WITH_PROXY=1 rebar3 shell

если тестовый эндпойнт доступен без прокси, то команда запуска:

rebar3 shell


Для тестирования использовался curl в терминале:

Получить все продукты
curl -i 'http://localhost:8886/assembled-products'

Получить продукт по его Id
curl -i 'http://localhost:8886/assembled-products?id=123'    curl to get ids 

Получить данные отфильтрованные по дате
curl -i 'http://localhost:8886/assembled-products?start_date=2026-04-16T05:41:18Z&end_date=2026-04-16T05:45:18Z'

Получить данные с фильтром по дате и по Id
curl -i 'http://localhost:8886/assembled-products?start_date=2026-04-16T07:00:18Z&end_date=2026-04-16T07:02:18Z&id=123'

Post запрос на изменение таймера опроса (начальное значение 30000) - передавать надо значение в мс
curl -X POST http://localhost:8886/assembled-products -H "Content-Type: application/json" -d '{"cmd": "set_timer", "value": 15000}'


Краткое описание:

Используется gen_server top_worker.erl в котором происходит запуск процессо на прослушивание сокета по порту 8886 (задефайнено в модуле). В модуле server.erl реализован сервер прослушивающий порт и обрабатывающий входные запросы. В модуле api_client.erl реализован походе в эндпойнт за данными.

В нем же происходит периодический тик таймера и обработка тиков,

Все АПИ запросы на эндпойнт приложения обрабатываться в handle_info.

Для хранения данных, которые получаем от внешнего эндпойнта с продуктами используются две ets таблицы, которые для простоты тоже созданы в init стадии ген_сервера

Таблицы - timestamps and products.

Поход за данными пришлось сделать с помощью либы hackney, потому что на рабочем месте где писал код, доступ до внешнего узла эндпойнта напрямую почему то не работает, пришлось через сокс5 прокси подключаться, а нативная httpc эрланга похоже не умеет через сокс5 ходить наружу.

Реализация вэб-сервера для управления таймеров и отдачей данных - сделал на базе gen_tcp, захотелось все это вспомнить, как там оно на базовом уровне).

Что не сделано:
- варнинги компиляции не трогал
- переменные аппликэйшн не использовал, для быстроты просто или в коде или через дефайны
- возможно где то не обрабатываю закрытие сокетов в нужном месте, но вроде удаленная сторона все подчищает
- разбивка по модулям - ну можно наверно и лучше
- не сделал еще привязку вида запрос - ответ - чтобы процессы корректно отвечали именно теми данными, которые от них запрашивали. но попытка была для этого использовать пиды, в handle_info ген сервера это в принципе видно
- минимум обработки негативных и каких то пограничных кейсов
- в коде остались местами io format - использовались для простого дебага, ну и для понимания что процесс опроса идет периодически

Если надо запустить приложение без прокси , то в модуле api_client.erl надо закомментировать строку 15 и расскоментировать 16 строку. Это в принципе можно через переменные окружения реализовать или еще добавить гибкости прир старте.
